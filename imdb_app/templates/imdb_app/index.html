Welcome to IMDB Site - Django based REST API

How to use this API?

    NORMAL USER
    • GET: https://imdbsite.herokuapp.com/user/ : This will GET all the movies in the database.

    • GET: https://imdbsite.herokuapp.com/user/?name=<movie_name> : This will GET all the movies in the database which contains name=movie_name.
      NOTE: Replace <movie_name> with your search string

    • POST, PUT and DELETE : These methods are not available for NORMAL USER

    ADMIN USER
    • For Admin access you need your user token. Steps are as follows:
      • Send a post request or Visit https://imdbsite.herokuapp.com/login/ : In Content box pass the username and password of admin as a JSON "{"username": "akash", "password": "akashswain"}"
        In Response a token will be returned --> "token": "6f1386dcbbf0f33554ff855d9ba971ccd01983dd"
      • Set this token in header of your GET, POST, DELETE requests.
        headers = {"Authorization": "6f1386dcbbf0f33554ff855d9ba971ccd01983dd"}

    • GET: https://imdbsite.herokuapp.com/adm/ : This will GET all the movies in the database. Don't forget to set the headers.

    • GET: https://imdbsite.herokuapp.com/user/?name=<movie_name> : This will GET all the movies in the database which contains name=movie_name. Don't forget to set the headers.
      NOTE: Replace <movie_name> with your search string

    • POST and PUT: Both POST and PUT requests are merged together. If no attribute is provided it will return an error - '"name" is required'.
      Now when you pass attribute "name" in your attributes list, then first it checks if it already exists in database,
      If YES, then it will update the Movie with other attributes like "popularity", "imdb_score", etc. if these are provided. No additional attribute are mandatory here.
      If NO, then it will create the Movie with the name provided. No additional attribute are mandatory here. If attributes are not provided it will be set else 0 or null or empty depending on attribute type.

      For testing use:
      • url = "https://imdbsite.herokuapp.com/adm/"

      • headers = {"Authorization": 'Token 6f1386dcbbf0f33554ff855d9ba971ccd01983dd'}

      • data = {
        "name": "Movie name",
        "imdb_score": 5,
        "popularity": 50,
        "director": "Director Name",
        "genre": ["genre1", "genre2", "genre3", "genre4", "genre5"]
        }

      Workflow/ Validations:
      • imdb_score and popularity: No number range validation.
      • genre: admin can add new genre but cannot update the existing one.

    • DELETE: DELETE request works on the exact movie name.

      For testing use: (Note: Replace <str:name> in below line with exact movie name.)
      • url= "https://imdbsite.herokuapp.com/adm/del_by_mov_name/<str:name>/"

      • headers = {"Authorization": 'Token 6f1386dcbbf0f33554ff855d9ba971ccd01983dd'}
